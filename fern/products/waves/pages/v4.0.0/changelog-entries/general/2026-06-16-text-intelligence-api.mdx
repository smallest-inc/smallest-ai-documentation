The Text Intelligence API analyses a piece of conversation text (a Pulse STT transcript, an inbound customer message, or any free-form prose) and returns structured insights in one call.

## What's new

`POST /waves/v1/text-intelligence` accepts a `text` string and a `features` object. Three independently opt-in features:

- **`topics`** — high-level conversation themes. Aggregate across many calls for trend tracking.
- **`intents`** — speaker intent (complaint, refund request, acceptance, etc.). Use for routing decisions or for tagging conversations downstream.
- **`sentiment`** — overall tone (`negative` | `neutral` | `positive`) plus a one-sentence English description of how sentiment evolved across the conversation.

The response only contains the fields you requested.

## Notes

- **Multilingual input** — Hindi, English, or mixed transcripts all work. Output topics/intents/sentiment are normalised to English so they're usable as cross-language dashboard labels.
- **Length cap** — 16,000 characters per request.
- **Auth + concurrency** — same `Authorization: Bearer $SMALLEST_API_KEY` and account-level concurrency limits as the rest of the Waves API.
