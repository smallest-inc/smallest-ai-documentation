---
title: Lightning v3.1 Pro — premium Indian female voices
---

Lightning v3.1 now has a **Pro** tier with 4 new Indian female conversational voices: **Meher**, **Saira**, **Kareena**, and **Rhea**. The Pro pool runs on dedicated inference capacity, delivering the same TTFB as standard Lightning v3.1 while keeping the premium voices isolated from general traffic.

**Languages supported:** English, Hindi (each voice handles natural code-switching between the two).

**How to use it:**

- **Atoms voice agents:** open the agent's voice picker and select the new **Pro** filter chip. Pick Meher / Saira / Kareena / Rhea. Your agent will automatically route to the Pro pool — no other configuration needed.
- **Lightning v3.1 API (direct):** pass `"model": "lightning_v3.1_pro"` in the request body alongside the chosen `voice_id`. The voices also appear under the standard v3.1 catalog, so existing integrations can pick them up immediately by `voice_id` and a body model flag.

**Voice availability:** the 4 Pro voices are **multimodel** — they show up under both Lightning v3.1 and the Pro filter in `GET /voice/get-all-models`. The `modelIds` field on each voice response lists every model the voice is callable from.

**Voice cloning:** not available on Lightning v3.1 Pro. Voice clones continue to use Lightning v3.1 (standard) and the existing voice-cloning flow. There is no migration required.
