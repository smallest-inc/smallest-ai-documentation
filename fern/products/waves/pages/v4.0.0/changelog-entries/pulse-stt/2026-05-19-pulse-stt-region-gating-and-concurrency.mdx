---
title: Pulse STT — region-disabled languages now return `LANGUAGE_NOT_ENABLED_IN_REGION` + concurrency caps tuned
date: 2026-05-19
description: Calls to offline languages that aren't enabled in your tenant's region now fail fast with a 400 and a clear error code, instead of routing to an unavailable pool.
---

Two cleanups on the Pulse STT gating layer:

**Region-disabled languages now return `400 LANGUAGE_NOT_ENABLED_IN_REGION`.** Previously, calls to an offline language not enabled in your tenant's region would route to an unavailable pool and time out. Now the request fails fast with a clear error:

```json
{
  "status": "error",
  "error_code": "LANGUAGE_NOT_ENABLED_IN_REGION",
  "message": "Language '<code>' is not enabled in this region. Contact support to request availability."
}
```

The region gate applies to both the pre-recorded REST endpoint and the streaming WebSocket path.

**Concurrency caps tuned.** Per-org concurrency limits on ASR have been recalibrated to match infrastructure capacity. Excess requests get `429`; back off with jitter.

**Migration:** if you're hitting `LANGUAGE_NOT_ENABLED_IN_REGION` for a language you previously had access to, contact your account manager — it's a region setting, not a code issue.
