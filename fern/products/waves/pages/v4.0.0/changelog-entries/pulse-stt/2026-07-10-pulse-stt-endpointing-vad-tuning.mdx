---
title: Pulse STT endpointing (VAD-driven finalization) + VAD tuning params
---

Pulse STT streaming now supports opt-in VAD-driven finalization on the WebSocket. Setting `endpointing=true` finalizes the current utterance when the acoustic VAD sees `endpointing_timeout` ms of trailing silence, off the model's ~1.1 s decode grid. On a 300 ms window this drops finalization latency from a variable ~0.9-2.2 s to a flat ~440 ms, with 0.00% WER change on internal benchmarks.

New query parameters (all opt-in, absent means unchanged behavior):

- `endpointing` (`true`/`false`, default `false`) — turn finalize-on-silence on
- `endpointing_timeout` (0-10000 ms, default 300) — trailing silence window; also shared with `vad_events`
- `vad_threshold` (0-1, default 0.5) — VAD speech-probability threshold
- `vad_min_speech_ms` (integer ms, default 120) — contiguous speech before the VAD begins tracking

The three tuning parameters (`endpointing_timeout`, `vad_threshold`, `vad_min_speech_ms`) also apply to [`vad_events`](/waves/documentation/speech-to-text-pulse/features/vad-events) when that is enabled. Both features share one VAD instance and one silence window per session.

The [VAD Events page](/waves/documentation/speech-to-text-pulse/features/vad-events) was updated to remove the previous "no tuning parameters" claim, which no longer holds.
