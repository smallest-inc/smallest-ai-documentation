title: Pulse STT — supported-language list updated to match current production
date: 2026-06-13
description: 'Pulse documentation now lists only the 15 individually-tested languages plus the two multi-language aggregators (`multi-eu`, `multi-indic`) that production accepts today.'
---

The Pulse STT supported-language list shown in the docs has been narrowed to match what the production API actually accepts in each region.

## Supported (15 individual + 2 aggregators)

| Language | Code |
|---|---|
| English | `en` |
| Hindi (with English code-switching) | `hi` |
| German | `de` |
| Spanish | `es` |
| Russian | `ru` |
| Italian | `it` |
| French | `fr` |
| Dutch | `nl` |
| Portuguese | `pt` |
| Mandarin | `zh` |
| Cantonese | `yue` |
| Japanese | `ja` |
| Korean | `ko` |
| Gujarati | `gu` |
| Marathi | `mr` |

Aggregators: `multi-eu` (en + de + es + it + pt + nl + ru + fr) and `multi-indic` (en + hi + gu + mr).

## Removed from the documented list

The following codes were previously listed but production today returns `400 LANGUAGE_NOT_ENABLED_IN_REGION` for them. They are removed from the docs to avoid first-try failures on copy-paste: Ukrainian, Polish, Czech, Slovak, Telugu, Oriya/Odia, Bengali, Latvian, Estonian, Romanian, Punjabi, Finnish, Swedish, Bulgarian, Tamil, Hungarian, Danish, Lithuanian, Maltese, Kannada, Malayalam, Tagalog, Indonesian, Malay. The `multi` and `multi-asian` aggregators are also removed.

## Why now

Customers running the documented codes for these languages were hitting HTTP 400 on the first request. Internal teams flagged that docs overstated coverage. The narrowed list matches Aditya's authoritative production-language set as of today.

## What you need to do

Existing code calling Pulse with one of the supported codes is unaffected. If you were calling one of the removed codes and hitting 400 errors, that was the underlying behavior — the docs just match it now. Reach out to support if you need a language enabled in your region.
