---
title: Legacy `lightning-asr` WebSocket reference removed from docs
---

The legacy `lightning-asr` WebSocket reference (`wss://api.smallest.ai/waves/v1/lightning/get_text`) has been removed from the docs. Pulse STT (`wss://api.smallest.ai/waves/v1/pulse/get_text`) is the supported real-time STT surface; the legacy endpoint was already unlinked from the v4 API reference navigation and was kept only for historical reference.

Files removed:

- `fern/products/waves/pages/v4.0.0/api-references/lightning-asr-ws.mdx` (and the `versions/v4.0.0` mirror).
- `fern/apis/waves/asyncapi/lightning-asr-ws.yaml` and its overrides.
- The corresponding generator entries in `fern/apis/waves/generators.yml` and `fern/apis/unified/generators.yml`.
- The `lightning-asr-ws.mdx` line in `scripts/.nav-ignore`.

If your code still calls `wss://api.smallest.ai/waves/v1/lightning/get_text`, migrate to Pulse STT — the request shape is similar, see [Pulse STT quickstart](/waves/documentation/speech-to-text-pulse/quickstart) and the [Pulse WebSocket reference](/waves/api-reference/api-reference/speech-to-text/speech-to-text).

→ [Pulse STT quickstart](/waves/documentation/speech-to-text-pulse/quickstart)
