The `keywords` query parameter on `wss://api.smallest.ai/waves/v1/stt/live` was missing from the unified-endpoint API reference. The parameter has always been accepted by the platform (verified live: intensifier `8` shifts the transcript from "Huss Miller" → "Hansi Muller"), but the unified-spec rewrite dropped its documentation.

Restored on both the base AsyncAPI spec and the v4 docs override, with the full intensifier guide, hallucination warning, and 100-keyword session limit.
