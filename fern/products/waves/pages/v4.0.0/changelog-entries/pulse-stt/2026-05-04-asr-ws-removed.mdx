---
title: Original `asr` WebSocket spec removed from SDK generation
---

The original AsyncAPI spec for `wss://api.smallest.ai/waves/v1/asr` has been removed. This is the third and final piece of the legacy STT WebSocket cleanup, after the `lightning-asr` WS removal in PR #110 and the `lightning-asr` HTTP removal in PR #112. **Pulse STT** (`wss://api.smallest.ai/waves/v1/pulse/get_text`) has been the supported real-time STT surface throughout; the `asr` channel was already unlinked from the v4 API reference navigation and was kept only for historical SDK generation.

Files removed:

- `fern/apis/waves/asyncapi/asr-ws.yaml` and its overrides.
- The corresponding generator entries in `fern/apis/waves/generators.yml` and `fern/apis/unified/generators.yml`.

SDK impact: the next SDK release will no longer expose the `transcribe_streaming_asr` / `receive_transcribe_streaming_asr` methods. Existing SDK releases keep working until upgrade. Migrate to Pulse STT — see the [Pulse STT WebSocket reference](/waves/api-reference/api-reference/speech-to-text/speech-to-text) and the [Pulse quickstart](/waves/documentation/speech-to-text-pulse/quickstart).
