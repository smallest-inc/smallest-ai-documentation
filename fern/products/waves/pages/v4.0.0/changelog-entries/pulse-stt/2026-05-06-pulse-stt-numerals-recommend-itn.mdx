---
title: Pulse STT — recommend `itn_normalize` over `numerals` for new integrations
---

The `numerals` query parameter on the Pulse STT WebSocket API still works and continues to behave as documented. For new integrations we now recommend [`itn_normalize=true`](/models/documentation/speech-to-text-pulse/features/inverse-text-normalization) instead — it covers digits as well as dates, currencies, phone numbers, and other spoken-form entities, and gives more consistent results across languages.

Existing code that uses `numerals` does not need to change.
