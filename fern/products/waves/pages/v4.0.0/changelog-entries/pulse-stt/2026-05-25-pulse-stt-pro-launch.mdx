<Warning>
Pulse Pro is **rolling out across regions** — at the time of publishing, the
`/waves/v1/speech/transcribe`, `/waves/v1/pulse-pro/get_text`, and
`/waves/v1/speech/deliver-webhook` routes are not yet live in every region
(probed 2026-05-26: returns 404 on `ap-south-1`). If you receive a 404,
contact your account manager for early-access enablement.
</Warning>

Pulse STT Pro is now available. It runs on a dedicated, higher-accuracy inference pool and is reachable via two new endpoints:

| Endpoint | Mode | Max file size |
|---|---|---|
| `POST /waves/v1/speech/transcribe` | Synchronous | 250 MB |
| `POST /waves/v1/speech/transcribe?webhook_url=…` | Async (webhook callback) | 50 MB |

The legacy `/waves/v1/pulse/*` routes are unchanged — Pulse Pro is a pure addition with its own pool, queue, and rate limits.

```bash
curl -X POST "https://api.smallest.ai/waves/v1/speech/transcribe?language=en" \
  -H "Authorization: Bearer $SMALLEST_API_KEY" \
  -H "Content-Type: audio/wav" \
  --data-binary @call.wav
```

**What you need to know:**

- Pulse Pro is **entitlement-gated**. The `feature-waves-stt-pro` plan feature must be enabled on your org. Standard plans include **10 minutes / month**, Enterprise plans **100 minutes / month**; metered usage is billed at $0.004 / minute.
- Per-org rate limit: **25 requests per minute** via the `feature-waves-stt-pro-rpm` cap. Bursts above this return `429`.
- Webhook delivery uses exponential backoff. The callback POSTs the transcript + request_id to your URL once the worker finishes.

**Migration:** no action — existing Pulse callers keep working. Opt in to Pro by switching to the new routes when you need the accuracy gain.
