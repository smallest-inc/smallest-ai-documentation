The Pre-Recorded Pulse STT API supports the following features:

## Available Features

<CardGroup cols={2}>
  <Card title="Word Timestamps" icon="clock" href="/models/documentation/speech-to-text-pulse/features/word-timestamps">
    Get precise timing information for each word in the transcription
  </Card>
  <Card title="Language Detection" icon="language" href="/models/documentation/speech-to-text-pulse/features/language-detection">
    Automatically detect the language of the audio
  </Card>
  <Card title="Diarization" icon="users" href="/models/documentation/speech-to-text-pulse/features/diarization">
    Identify and label different speakers in the audio
  </Card>
  <Card title="Gender Detection" icon="user" href="/models/documentation/speech-to-text-pulse/features/gender-detection">
    Predict speaker gender alongside transcription
  </Card>
  <Card title="Emotion Detection" icon="face-smile" href="/models/documentation/speech-to-text-pulse/features/emotion-detection">
    Detect emotional tone in the transcribed speech
  </Card>
  <Card title="Redaction" icon="eye-slash" href="/models/documentation/speech-to-text-pulse/features/redaction">
    Automatically redact sensitive information from transcriptions
  </Card>
  <Card title="Utterances" icon="comment" href="/models/documentation/speech-to-text-pulse/features/utterances">
    Segment transcription into meaningful utterances (requires word_timestamps)
  </Card>
</CardGroup>

<Note>
**Keyword Boosting** is available on the [Real-Time WebSocket API](/models/documentation/speech-to-text-pulse/realtime-web-socket/features) only. It is not supported on the pre-recorded HTTP endpoint.
</Note>
