Query parameters that the platform accepts but the API reference no longer documented are restored. Each was live-verified end-to-end before adding back; the WS set and REST set are scoped per protocol per the platform's Zod schemas (`lightningAsrQuerySchema` vs `lightningAsrWebsocketQuerySchema`).

## `WSS /waves/v1/stt/live` — 10 params restored

| Parameter | Verified behavior |
|---|---|
| `format` | Master formatting switch — `false` disables punctuate, capitalize, and ITN at once. |
| `punctuate` | `false` strips `.`, `,`, `?`, `!` from finals + `words[]` + `utterances[]`. |
| `capitalize` | `false` lowercases the entire transcript output. |
| `itn_normalize` | `true` converts `five five five one two three` → `5551234` in finals. |
| `finalize_on_words` | `false` disables automatic word-count-based finalization. |
| `max_words` | Caps each `is_final` segment at the given word count. |
| `redact_pii` | Replaces names / phones / addresses with `[FIRSTNAME_*]`, `[PHONENUMBER_*]`, `[ADDRESS_*]`. |
| `redact_pci` | Replaces card-like numbers with `[ACCOUNTNUMBER_*]`. |
| `sentence_timestamps` | Adds a populated `utterances[]` array to each transcription event. |
| `full_transcript` | Each event carries a cumulative `full_transcript` field across the session. |

`utterances[]` was also added to the documented WS response shape since `sentence_timestamps=true` populates it.

## `POST /waves/v1/stt/` — 4 params restored

Only params with verified visible behavior on the pre-recorded REST endpoint were added back. (`punctuate`, `capitalize`, `keywords`, and `numerals` are accepted at the wire level on REST but did not change the transcript in our single-utterance live test — held back pending a more thorough test matrix.)

| Parameter | Verified behavior |
|---|---|
| `redact_pii` | Names, phone numbers, addresses replaced with bracketed tokens. |
| `redact_pci` | Card-like numbers replaced with `[ACCOUNTNUMBER_*]`. |
| `emotion_detection` | Response adds an `emotions` object (label → confidence). |
| `gender_detection` | Response adds a `gender` field with the detected label. |

`emotions` and `gender` were added to the documented `TranscriptionResponse` shape.
