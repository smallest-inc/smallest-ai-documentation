Lightning v3.1 (Standard) and Lightning v3.1 Pro are evaluated on the [EmergentTTS benchmark](https://huggingface.co/datasets/bosonai/EmergentTTS) — 1,088 listener-judged samples spanning English prose, multilingual code-switching, expressive delivery, paralinguistic content, and stress tests for prosody. Scores below come from internal evaluation using the same dataset, scoring rubric, and LLM-as-a-Judge framework across all models. See [Metrics Overview](/waves/documentation/text-to-speech-lightning/benchmarks/metrics-overview) for what each metric measures, and [Evaluation Walkthrough](/waves/documentation/text-to-speech-lightning/benchmarks/evaluation-walkthrough) to reproduce these results in your own environment.

## Latency

### Time-to-First-Byte (TTFB)

TTFB measures the wall-clock delay between sending the synthesis request and the first audio byte arriving on the wire. Lower is better for real-time and conversational use cases.

### Real-Time Factor (RTF)

`RTF = Audio Duration ÷ Processing Time`. Values above 1.0 mean the model produces audio faster than playback. Both Standard and Pro run at **3.3× real-time** on NVIDIA L40S, so a 10-second utterance is fully synthesized in ~3 seconds.

## When to use which model

| Need | Recommended model |
|---|---|
| Lowest cost, broadest language coverage, voice cloning | **Lightning v3.1 (Standard)** — `"model": "lightning_v3.1"` or omit |
| Premium voice quality on a curated catalog, best Emotions / Pause Placement / WV-MOS | **Lightning v3.1 Pro** — `"model": "lightning_v3.1_pro"` |
| Mixing both in one app | Either model is selected per request via the `model` body field on the unified `/waves/v1/tts*` endpoints — no separate endpoint required |

## Optimization tips

- **Stream over WebSocket** for the lowest customer-perceived latency. TTFB ~200 ms is the WS path; sync HTTP `POST /tts` waits for the full clip before responding.
- **Pin proper nouns** with [pronunciation dictionaries](/waves/documentation/text-to-speech-lightning/pronunciation-dictionaries) — fastest way to close the residual WER gap.
- **Keep chunks ≤ 250 chars** (140 chars optimal). Longer inputs serialize and add latency.
- **Pick a voice trained on your target language.** Each voice has a `tags.language` field on the catalog response; passing a `language` outside that list still synthesizes but produces accented or mis-pronounced output.
