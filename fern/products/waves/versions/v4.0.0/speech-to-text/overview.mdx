The Speech-to-Text API transcribes audio via the unified endpoint `https://api.smallest.ai/waves/v1/stt/`. The model is selected via the `?model=` query parameter:

- **`?model=pulse`**: multilingual, supports streaming and pre-recorded transcription.
- **`?model=pulse-pro`**: leaderboard-ranked English accuracy (5.42% ESB avg WER, tied #2 on the public Open ASR Leaderboard). Pre-recorded HTTP only; no streaming worker yet.

Live streaming runs on `WS /waves/v1/stt/live?model=pulse`. Pulse Pro on the live path returns `400` with a directive to use HTTP.

## Transcription Modes

We offer two transcription modes to cover a wide range of use cases. Choose the one that best fits your needs:

<CardGroup cols={2}>
  <Card title="Pre-Recorded" icon="file-audio" href="/waves/documentation/speech-to-text-pulse/pre-recorded/quickstart">
    Transcribe audio files using synchronous HTTPS POST requests. Perfect for batch processing, archived media, and offline transcription workflows.
  </Card>
  <Card title="Real-Time" icon="broadcast-tower" href="/waves/documentation/speech-to-text-pulse/realtime-web-socket/quickstart">
    Stream audio and receive transcription results as the audio is processed. Ideal for live conversations, voice assistants, and low-latency applications.

## Feature highlights

Our models specialize in processing audio to preserve information that is often lost during conventional speech to text conversion.

## Supported languages

Pulse currently supports the same core set of languages in both transcription modes. The tables below mirror the [Pulse model card](/waves/model-cards/speech-to-text/pulse) — keep it as the source of truth.

### Streaming (Real-Time, WebSocket)

Use these codes on `WS /waves/v1/stt/live?model=pulse`. For unknown audio, pick the regional aggregator: `multi-eu` (European set), `multi-indic` (Indic set), or `multi-asian` (East Asian set, US region only).

| Language | Code |
|---|---|
| English | `en` |
| Hindi (+ English code-switching) | `hi` |
| German | `de` |
| Spanish | `es` |
| Russian | `ru` |
| Italian | `it` |
| French | `fr` |
| Dutch | `nl` |
| Portuguese | `pt` |
| Mandarin | `zh` &mdash; US region only \[1] |
| Cantonese | `yue` &mdash; US region only \[1] |
| Japanese | `ja` &mdash; US region only \[1] |
| Korean | `ko` &mdash; US region only \[1] |
| Gujarati | `gu` |
| Marathi | `mr` |
| Multi-Indic aggregator | `multi-indic` &mdash; auto-detects across `en`, `hi`, `gu`, `mr` |
| Multi-EU aggregator | `multi-eu` &mdash; auto-detects across `en`, `de`, `es`, `it`, `pt`, `nl`, `ru`, `fr` |
| Multi-Asian aggregator | `multi-asian` &mdash; auto-detects across `en`, `ja`, `ko`, `zh`, `yue`; US region only \[1] |

\[1] `ja`, `yue`, `zh`, `ko`, and the `multi-asian` aggregator are served from the US region only. Connect to `wss://api.us.smallest.ai/waves/v1/stt/live?model=pulse` (or the legacy `wss://api.us.smallest.ai/waves/v1/pulse/get_text`) instead of the default `wss://api.smallest.ai/...` host. Requesting them on the default host closes the connection without a transcription frame.

### Non-Streaming (Pre-Recorded, HTTP)

Use these codes on `POST /waves/v1/stt/?model=pulse`.

| Language | Code |
|---|---|
| English | `en` |
| Hindi (+ English code-switching) | `hi` |
| German | `de` |
| Spanish | `es` |
| Russian | `ru` |
| Italian | `it` |
| French | `fr` |
| Dutch | `nl` |
| Portuguese | `pt` |
| Mandarin | `zh` &mdash; accepted but unreliable \[2] |
| Cantonese | `yue` &mdash; not supported on batch (returns `400 LANGUAGE_NOT_ENABLED_IN_REGION`) |
| Japanese | `ja` &mdash; accepted but unreliable \[2] |
| Korean | `ko` &mdash; accepted but unreliable \[2] |
| Gujarati | `gu` |
| Marathi | `mr` |
| Multi-Indic aggregator | `multi-indic` &mdash; auto-detects across `en`, `hi`, `gu`, `mr` |
| Multi-EU aggregator | `multi-eu` &mdash; auto-detects across `en`, `de`, `es`, `it`, `pt`, `nl`, `ru`, `fr` |

\[2] `ja`, `zh`, `ko`, and the `multi-asian` aggregator are accepted by batch (HTTP 200) but transcripts are unreliable — the batch model lacks proper checkpoints for East Asian audio and tends to return garbled CJK-mixed output. Cantonese (`yue`) is the one exception: batch returns `400 LANGUAGE_NOT_ENABLED_IN_REGION`. For accurate Asian-language transcription, use the streaming endpoint on `wss://api.us.smallest.ai/waves/v1/stt/live`.
