When speech recognition drops a caller's reply, your agent no longer sits in awkward silence — it now notices the miss and naturally asks the caller to repeat themselves. This is on by default for every agent, using prompts that adapt to the conversation's language. Prefer your own wording? Add custom re-prompt responses under Speech settings and the agent will use those instead.

```
{
  "sttMissDetectorConfig": {
    "isEnabled": true,
    "repeatPromptMessages": ["Sorry, I didn't catch that — could you say it again?"]
  }
}
```

**Migration:** no action — existing agents get the default (enabled, built-in prompts) automatically
