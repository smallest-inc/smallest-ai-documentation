---
title: Post-call tagging engine — rule-based call categorisation
date: 2026-05-25
description: Define rules on transcripts and disposition fields to auto-tag completed calls. Tags flow through to analytics and webhook payloads.
---

A new **post-call tagging engine** auto-categorises completed calls based on rules you configure on the agent. Tags are computed from the transcript, disposition metrics, and call metadata, and are visible in analytics + the call detail page.

**What you can build:**

- Rules like *"if transcript contains 'cancel my subscription' AND disposition.outcome != 'resolved'"* → tag the call `churn-risk`.
- Multiple rules per agent; tags accumulate.
- Tags ship in the post-call webhook payload under `tags: ["…", "…"]` so downstream systems can route on them.

**How to enable:**

1. Open the agent in the Atoms console.
2. New **Post-call tags** panel under the agent config.
3. Add one or more rules — each is a boolean expression over transcript / disposition fields with a target tag name.

**Migration:** no action — additive. No tags computed unless you configure rules.
