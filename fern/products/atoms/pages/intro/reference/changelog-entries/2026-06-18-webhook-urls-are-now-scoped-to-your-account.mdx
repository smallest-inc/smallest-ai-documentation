Webhook endpoint URLs are now scoped to your account instead of being unique across the whole platform. You can register a webhook URL even if another account already uses the same endpoint — useful when several teams route events through a shared integration. A single account still can't register the exact same URL twice. Creating a webhook now also checks that the endpoint is a valid URL, so a typo is rejected up front instead of silently failing when an event fires.
