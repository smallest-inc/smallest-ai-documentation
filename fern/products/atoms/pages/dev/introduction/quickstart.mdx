Install the SDK, write your first agent crew, and test it — locally or deployed to the cloud.

## Prerequisites

<Note>
  **Smallest API key.** Get one from [app.smallest.ai/dashboard/api-keys](https://app.smallest.ai/dashboard/api-keys) and export it:
  ```bash
  export SMALLEST_API_KEY=sk_...
  ```
  Used by the CLI (`smallestai auth login`) and the SDK helper classes (`Call`, `KB`, `Campaign`, `Audience`).
</Note>

<Note>
  **An LLM API key.** Pick one — the examples below show both:
  - **Electron** (Smallest's own LLM, OpenAI-compatible) — reuses your `SMALLEST_API_KEY`. No extra key needed.
  - **OpenAI** — `export OPENAI_API_KEY=sk-...` if you'd rather use GPT models.
</Note>

## Installation

```bash
pip install smallestai
```

## Write Your First Agent Crew

Create two files: one for the crew node logic, and one to run the server.

<Steps>
  <Step title="Create assistant.py">
    Subclass `OutputCrewNode` and implement `generate_response()` to stream LLM output.

    <Tabs>
      <Tab title="Electron (Smallest LLM)">
        Reuses your `SMALLEST_API_KEY` — no extra key needed. `OpenAIClient` takes any OpenAI-compatible endpoint via `base_url`.

        ```python assistant.py
        import os
        from dotenv import load_dotenv
        from smallestai.atoms.crew.nodes import OutputCrewNode
        from smallestai.atoms.crew.clients.openai import OpenAIClient

        load_dotenv()  # picks up keys from .env locally AND on the cloud

        class Assistant(OutputCrewNode):
            def __init__(self):
                super().__init__(name="assistant")
                self.llm = OpenAIClient(
                    model="electron",
                    api_key=os.getenv("SMALLEST_API_KEY"),
                    base_url="https://api.smallest.ai/waves/v1",
                )

            async def generate_response(self):
                response = await self.llm.chat(
                    messages=self.context.messages,
                    stream=True
                )
                async for chunk in response:
                    if chunk.content:
                        yield chunk.content
        ```
      </Tab>
      <Tab title="OpenAI">
        ```python assistant.py
        import os
        from dotenv import load_dotenv
        from smallestai.atoms.crew.nodes import OutputCrewNode
        from smallestai.atoms.crew.clients.openai import OpenAIClient

        load_dotenv()  # picks up keys from .env locally AND on the cloud

        class Assistant(OutputCrewNode):
            def __init__(self):
                super().__init__(name="assistant")
                self.llm = OpenAIClient(
                    model="gpt-4o-mini",
                    api_key=os.getenv("OPENAI_API_KEY"),
                )

            async def generate_response(self):
                response = await self.llm.chat(
                    messages=self.context.messages,
                    stream=True
                )
                async for chunk in response:
                    if chunk.content:
                        yield chunk.content
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Create server.py">
    Wire up `AtomsCrewApp` with a `setup_handler` that adds your crew node to the session.

    ```python server.py
    from smallestai.atoms.crew.server import AtomsCrewApp
    from smallestai.atoms.crew.session import CrewSession
    from assistant import Assistant

    async def on_start(session: CrewSession):
        session.add_node(Assistant())
        await session.start()
        await session.wait_until_complete()

    if __name__ == "__main__":
        app = AtomsCrewApp(setup_handler=on_start)
        app.run()
    ```

    <Tip>
      Your entry point can be named anything (`run.py`, `serve.py`, etc.). When deploying, specify it with `--entry-point your_server.py`.
    </Tip>

    <Tip>
      **Want a greeting?** Use `@session.on_event` to speak when the user joins:
      ```python
      @session.on_event("on_event_received")
      async def on_event(_, event):
          if isinstance(event, SDKSystemUserJoinedEvent):
              agent.context.add_message({"role": "assistant", "content": "Hello!"})
              await agent.speak("Hello! How can I help?")
      ```
      Adding the greeting to context ensures the LLM knows the conversation has started.
    </Tip>
  </Step>

  <Step title="Create requirements.txt">
    The cloud build installs from this file. Pin `smallestai` and any libs your code imports:

    ```txt requirements.txt
    smallestai>=4.4.4
    loguru>=0.7.0
    ```
  </Step>
</Steps>

## Run Your Agent Crew

Once your files are ready, you have two options:

<Tabs>
  <Tab title="Run Locally">
    For development and testing, run the file directly:

    ```bash
    python server.py
    ```

    This starts a WebSocket server on `localhost:8080`. In a separate terminal, connect to it:

    ```bash
    smallestai agent-crew chat
    ```

    No account or deployment needed.
  </Tab>

  <Tab title="Deploy to Platform">
    To have Smallest AI host your agent in the cloud (for production, API access, or phone calls). Deploy has two halves:

    1. **The agent record** on the platform — name, voice, language, `first_message`, attached phone numbers. Created via SDK or dashboard.
    2. **The agent code** (your `assistant.py` + `server.py`) — uploaded as a zip and run as your agent's "brain" during calls.

    The CLI links them via `agent-crew init` and uploads the code via `agent-crew deploy`.

    <Steps>
      <Step title="Create the agent record (SDK)">
        ```python
        from smallestai import SmallestAI
        import os

        # Pass token= explicitly — the SmallestAI client otherwise looks for
        # SMALLEST_AI_TOKEN, while the CLI and helpers use SMALLEST_API_KEY.
        client = SmallestAI(token=os.getenv("SMALLEST_API_KEY"))

        response = client.atoms.agents.create_a_new_agent(name="my-test-agent")
        agent_id = response.data   # response.data is the agent id, a string
        print(agent_id)
        ```

        Save the printed `agent_id` — you'll need it in the next step.

        <Note>
          **The greeting comes from your `server.py` code, not the agent record.** Passing `first_message=...` to `create_a_new_agent` currently does not persist on the platform (verified 2026-05-20). Use the `SDKSystemUserJoinedEvent` hook in `server.py` (shown above in the *Create server.py* step) to make your agent speak first.
        </Note>
      </Step>

      <Step title="Authenticate the CLI">
        ```bash
        smallestai auth login
        ```
        Paste your `SMALLEST_API_KEY` when prompted, or pipe it in:
        ```bash
        echo $SMALLEST_API_KEY | smallestai auth login
        ```
        Credentials are stored at `~/.smallestai/credentials.json` (`0600`).
      </Step>

      <Step title="Link local code to the agent record">
        ```bash
        smallestai agent-crew init
        ```

        Interactive picker — choose the agent you just created. It writes one file: `.smallestai/config.toml`.

        <Tip>
          **No TTY?** Write the file yourself:
          ```toml .smallestai/config.toml
          agent_id = "<the-id-you-saved>"
          ```
        </Tip>
      </Step>

      <Step title="Ship your env vars in a .env">
        <Warning>
          Platform-side secrets injection **isn't built yet.** The deploy zip does not carry your shell environment.
        </Warning>

        **How `.env` works on the cloud**:
        - The deploy zip ships every file in your project dir, except for `__pycache__/`, `.venv/`, etc. `.env` is **not** excluded.
        - The cookbook's `assistant.py` calls `load_dotenv()` at import time → on the cloud, that reads `.env` from the same directory.
        - End result: keys defined in your local `.env` are available via `os.getenv(...)` inside the running cloud agent.

        Create one next to `server.py`:

        ```bash
        cat > .env <<EOF
        OPENAI_API_KEY=$OPENAI_API_KEY
        SMALLEST_API_KEY=$SMALLEST_API_KEY
        EOF
        ```

        <Warning>
          **Skip this and the deployed agent boots fine but stays silent on calls.** `OpenAIClient(api_key=None)` only logs a warning to cloud logs (which you can't see), then fails on the first LLM request — no traceback in front of the caller, just silence.
        </Warning>
      </Step>

      <Step title="Deploy">
        ```bash
        smallestai agent-crew deploy --entry-point server.py
        ```

        Zips your dir, uploads it, builds, deploys. Takes ~30s. Output ends with a build id.
      </Step>

      <Step title="Make Live">
        <Warning>
          Deploys are **not live by default.** Builds land in `SUCCEEDED` with `isLive: null`. Your agent won't answer calls until you promote one.
        </Warning>

        ```bash
        smallestai agent-crew builds
        ```

        Select your build, choose **Make Live**.

        <Tip>
          **No TTY?** `agent-crew builds` needs an interactive terminal. Promote via REST instead:

          ```bash
          curl -X PATCH "https://api.smallest.ai/atoms/v1/sdk/agents/$AGENT_ID/builds/$BUILD_ID" \
            -H "Authorization: Bearer $SMALLEST_API_KEY" \
            -H "Content-Type: application/json" \
            -d '{"isLive": true}'
          ```

          Successful response: `{"status": true, "data": {"message": "Build set as live successfully"}}`.
        </Tip>
      </Step>

      <Step title="Attach a phone number to take calls">
        New agents have no phone number. Attach one from the [Atoms dashboard](https://app.smallest.ai) (SIP/Twilio/Telnyx/Vonage) or via `client.atoms.phone_numbers`. See [Phone Numbers](/atoms/developer-guide/build/phone-calling/overview).
      </Step>
    </Steps>
  </Tab>
</Tabs>
