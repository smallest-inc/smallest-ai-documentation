The `smallestai agent-swarm` CLI is built for rapid iteration. Test your swarm locally with a single command, then deploy to the cloud when ready. No configuration files, no Docker, no infrastructure management.

<Info>
  **Local-first development**: Run and test your agent entirely on your machine. Deploy only when you're satisfied.
</Info>

---

## Installation

The CLI comes bundled with the SDK:

```bash
pip install smallestai
```

Verify the installation:

```bash
smallestai --help
```

You'll also need an OpenAI API key for the example below — export it before running:

```bash
export OPENAI_API_KEY=sk-...
```

## Create Your First Agent Swarm

Before the CLI has anything to connect to, you need a Python file that defines a swarm and starts a WebSocket server. The fastest path is to copy the [`getting_started`](https://github.com/smallest-inc/cookbook/tree/main/voice-agents/getting_started) example from our cookbook — a minimal, single-node swarm that streams chat completions from an LLM and greets the caller on join.

```bash
# Grab the two files into a directory of your choice
curl -O https://raw.githubusercontent.com/smallest-inc/cookbook/main/voice-agents/getting_started/server.py
curl -O https://raw.githubusercontent.com/smallest-inc/cookbook/main/voice-agents/getting_started/assistant.py
```

`server.py` is the entry point — it boots an `AtomsSwarmApp` on `ws://localhost:8080/ws`. `assistant.py` contains the `OutputSwarmNode` subclass that streams the LLM response; extend it with your own prompt, tools, and logic.

For a richer example with a background node running sentiment analysis alongside the main agent, see [`background_agent`](https://github.com/smallest-inc/cookbook/tree/main/voice-agents/background_agent) in the same cookbook.

<Info>
  **What you're working with:**
  - `OutputSwarmNode` — the base class for nodes that produce user-facing output. Override `generate_response()` to define what the agent says.
  - `SwarmSession` — the runtime that owns the WebSocket connection and runs your nodes for each call.
  - `AtomsSwarmApp` — the FastAPI WebSocket server that the CLI's `chat` command, and the production Atoms orchestrator, connect to.
</Info>

## Local Development Workflow

The CLI's best feature is instant local testing. No need to deploy to test changes.

<Steps>
  <Step title="Start Your Agent Server">
    Run the entry point to start a local WebSocket server:
    
    ```bash
    python server.py
    ```
    
    This spins up a server on `localhost:8080` that mimics the production environment.
  </Step>
  
  <Step title="Connect via CLI">
    In another terminal, start an interactive voice session:
    
    ```bash
    smallestai agent-swarm chat
    ```
    
    The CLI connects to your local server and lets you converse with your agent in real-time.
    
    ![CLI Chat](../../images/cli-chat.png)
  </Step>
  
  <Step title="Iterate Rapidly">
    Make code changes, restart the server, and reconnect with `chat`. No redeploys needed.
  </Step>
</Steps>

## Cloud Deployment

When you're ready for production, deploy to Smallest AI's managed infrastructure.

<Warning>
  **Prerequisite:** You must first create an agent on the [Atoms platform](https://atoms.smallest.ai) before deploying. The CLI links your local code to an existing platform agent.
</Warning>

<Steps>
  <Step title="Authenticate">
    ```bash
    smallestai auth login
    ```
    
    Opens a browser for OAuth. Your credentials are stored locally.
  </Step>
  
  <Step title="Link to Platform Agent">
    ```bash
    smallestai agent-swarm init
    ```
    
    Select which agent from [atoms.smallest.ai](https://atoms.smallest.ai) to link your local code to.
  </Step>
  
  <Step title="Deploy">
    ```bash
    smallestai agent-swarm deploy --entry-point server.py
    ```
    
    Packages your code and pushes it to the cloud. Takes about 30 seconds.
  </Step>
  
  <Step title="Go Live">
    ```bash
    smallestai agent-swarm builds
    ```
    
    Select your build and choose **Make Live** to start serving traffic.
  </Step>
</Steps>

## Command Reference

### Authentication

| Command | Description |
|---------|-------------|
| `smallestai auth login` | Authenticate with your Smallest AI account |
| `smallestai auth logout` | Clear stored credentials |

### Agent Management

| Command | Description |
|---------|-------------|
| `smallestai agent-swarm init` | Link local directory to a platform agent |
| `smallestai agent-swarm deploy` | Deploy code to the cloud |
| `smallestai agent-swarm builds` | View and manage deployments |
| `smallestai agent-swarm chat` | Start interactive session with local agent |

### Common Options

| Option | Description |
|--------|-------------|
| `--entry-point <file>` | Specify the main Python file (default: `server.py`) |
| `--help` | Show help for any command |

## Build Management

Deployments are not live by default. This gives you a safety buffer.

<Warning>
  **One Live Build Per Agent**: Making a new build live automatically takes down the previous one.
</Warning>

**To promote a build:**
1. Run `smallestai agent-swarm builds`
2. Select the desired build
3. Choose **Make Live**

**To roll back:**
1. Run `smallestai agent-swarm builds`
2. Select the previous build
3. Choose **Make Live**

**To take down completely:**
1. Run `smallestai agent-swarm builds`
2. Select the **LIVE** build
3. Choose **Take Down**
