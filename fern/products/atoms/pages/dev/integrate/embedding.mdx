Two paths to put a voice agent inside your own product:

1. **Integrate** — wire the Atoms WebSocket protocol into your app yourself, using the platform's native APIs. More code to own, more control.
2. **Embed** — drop in a pre-built component or fork a finished reference app. Fastest to ship.

Which one you pick depends on how much of the UX you want to control and whether a first-party SDK exists for your stack. The table below maps each platform to both paths.

## At a glance

| Platform | Embed (pre-built) | Integrate (build your own) |
|---|---|---|
| **Web**         | [`<atoms-widget>` script tag](#web) (today) | [WebSocket SDK](/atoms/developer-guide/integrate/websocket-sdk) |
| **React Native**| [Hearthside cookbook](https://github.com/smallest-inc/cookbook/tree/main/voice-agents/react_native_voice_agent) — fork + customize | [React Native guide](/atoms/developer-guide/integrate/mobile/react-native) |
| **iOS (Swift)** | [iOS Swift cookbook](https://github.com/smallest-inc/cookbook/tree/main/voice-agents/ios_swift_voice_agent) | [iOS Swift guide](/atoms/developer-guide/integrate/mobile/ios-swift) |
| **Android (Kotlin)** | [Android Kotlin cookbook](https://github.com/smallest-inc/cookbook/tree/main/voice-agents/android_kotlin_voice_agent) | [Android Kotlin guide](/atoms/developer-guide/integrate/mobile/android-kotlin) |
| **Flutter**     | [Flutter cookbook](https://github.com/smallest-inc/cookbook/tree/main/voice-agents/flutter_voice_agent) | [Flutter guide](/atoms/developer-guide/integrate/mobile/flutter) |

Drop-in SDK packages for iOS / Android / RN / Flutter are on the roadmap. Until then, the cookbooks are the "embed" path for mobile — fork, swap credentials, ship.

## Web

Paste two lines into any HTML page:

```html
<atoms-widget assistant-id="YOUR_AGENT_ID"></atoms-widget>
<script src="https://unpkg.com/atoms-widget-core@latest/dist/embed/widget.umd.js"></script>
```

That renders a floating call bubble on the page. Clicking it opens a voice session with your agent. The component is a web-standard Custom Element; works in any framework (React, Vue, Svelte, plain HTML). Configuration — theme, position, agent id — goes through HTML attributes.

To customize the widget's appearance or placement, see the [Widget features reference](/atoms/atoms-platform/features/widget).

## React Native

No npm package yet. For a full working app you can fork today, use the [Hearthside](https://github.com/smallest-inc/cookbook/tree/main/voice-agents/react_native_voice_agent) cookbook. It's a ~600-line Expo app covering:

- Realtime WebSocket client with reconnect
- Mic capture + gapless PCM playback using `react-native-audio-api`
- In-app settings sheet that drives the full agent-update REST flow (voice, speed, language)
- Mute toggle, transport diagnostics, correct iOS audio session for full-speaker playback

Fork it, replace the narrator prompt with your own agent, and ship.

If you want to wire the WebSocket into an existing RN app rather than fork a new one, follow the [React Native integration guide](/atoms/developer-guide/integrate/mobile/react-native) for a drop-in protocol reference.

## iOS (Swift)

Same pattern, native Swift. The [iOS Swift cookbook](https://github.com/smallest-inc/cookbook/tree/main/voice-agents/ios_swift_voice_agent) ships a SwiftUI app built on `URLSessionWebSocketTask` + `AVAudioEngine`, with no external dependencies. Xcode project is generated by `xcodegen` from a single `project.yml`, so you can audit the full build config at a glance.

For integrating into an existing app see the [iOS Swift guide](/atoms/developer-guide/integrate/mobile/ios-swift).

## Android (Kotlin)

[Android Kotlin cookbook](https://github.com/smallest-inc/cookbook/tree/main/voice-agents/android_kotlin_voice_agent) — Jetpack Compose app using OkHttp's WebSocket and platform `AudioRecord` / `AudioTrack`. Gradle build with `BuildConfig` injection for credentials, `minSdk 24`.

For integrating into an existing app see the [Android Kotlin guide](/atoms/developer-guide/integrate/mobile/android-kotlin).

## Flutter

[Flutter cookbook](https://github.com/smallest-inc/cookbook/tree/main/voice-agents/flutter_voice_agent) — Material 3 app on top of `web_socket_channel`, `mic_stream`, and `flutter_pcm_sound`. Credentials injected via `--dart-define` at build time.

For integrating into an existing app see the [Flutter guide](/atoms/developer-guide/integrate/mobile/flutter).

## Cross-platform testing guide

Each cookbook has a README with platform-specific setup. The repo also ships a [`MOBILE_COOKBOOKS.md`](https://github.com/smallest-inc/cookbook/blob/main/voice-agents/MOBILE_COOKBOOKS.md) cross-reference — validation checklists, shared debug patterns (transport counter, mute-for-loop-debugging, Android emulator mic toggle, iOS simulator audio caveat), and a "when to use which cookbook" matrix.

## Roadmap

First-party mobile SDKs wrapping the cookbook patterns as one-line components:

```tsx
// React Native (forthcoming)
import { AtomsWidget } from '@smallest-ai/atoms-react-native';
<AtomsWidget apiKey={K} agentId={ID} mode="floating" theme={{ primary: '#DB984A' }} />
```

```swift
// iOS (forthcoming)
import SmallestAtoms
SmallestAtomsView(apiKey: K, agentId: ID)
```

```kotlin
// Android (forthcoming)
AtomsWidget(apiKey = K, agentId = ID)
```

```dart
// Flutter (forthcoming)
AtomsWidget(apiKey: K, agentId: ID)
```

Each will wrap the corresponding cookbook — the transport, audio, and UI code you see in the cookbooks today becomes the SDK tomorrow.
